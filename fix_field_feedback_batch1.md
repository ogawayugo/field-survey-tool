## 1. マーカーの対象点を自由にドラッグ移動

### 現状の問題

マーカーを一度配置すると、対象点（矢印先の点）の位置を変更できない。位置がズレていた場合に修正できず、削除して再作成する必要があった。

### 改善

通常の点マーカー（type='point'）について、対象点をドラッグして自由に移動できるようにする。

- マーカー選択中（`selectedMarkerId === marker.id` の状態）に、対象点の◯にドラッグハンドルとしての機能を付与
- ドラッグで `marker.x`, `marker.y` を更新
- テキストボックスは中点を追従（既存ロジックそのまま：テキストボックスは `(x, y)` から上方向に固定オフセット）
- 矢印も自動で追従（既存ロジックそのまま）
- 写真の端でclamp（0.0〜1.0）

### 実装ガイド

`MarkerOverlay.jsx` の通常マーカー（点）描画部分：

```jsx
function PointMarker({ marker, isSelected, onUpdate, ... }) {
  // ... 既存のテキストボックスと矢印描画 ...
  
  // 対象点：選択中ならドラッグ可能
  return (
    <>
      {/* テキストボックス（既存）*/}
      {/* 矢印（既存）*/}
      
      {/* 対象点（ドラッグ可能に） */}
      <div
        style={{
          position: 'absolute',
          left: `${marker.x * 100}%`,
          top: `${marker.y * 100}%`,
          transform: 'translate(-50%, -50%)',
          width: isSelected ? 16 : 8,  // 選択中は大きめでドラッグしやすく
          height: isSelected ? 16 : 8,
          borderRadius: '50%',
          backgroundColor: partColor,
          border: isSelected ? '2px solid white' : 'none',
          cursor: isSelected ? 'grab' : 'pointer',
          touchAction: 'none',
          zIndex: isSelected ? 25 : 15,
        }}
        onPointerDown={(e) => {
          if (!isSelected) return;
          e.preventDefault();
          // ドラッグ開始
          startDrag(e, (newX, newY) => {
            onUpdate({ ...marker, x: newX, y: newY });
          });
        }}
      />
    </>
  );
}
```

範囲マーカー（type='range'）は既にドラッグハンドルを持っているので変更不要。

### 動作確認

- [ ] マーカーをタップ → 選択状態（対象点が大きくなる）
- [ ] その対象点をドラッグ → 自由に移動できる
- [ ] テキストボックスは中点を追従して動く
- [ ] 矢印も追従する
- [ ] 写真の端でclampされる

## 2. 所見欄の復活

### 配置

特記事項欄の上、判定理由（外観診断）の下あたりに新規セクション「所見」を追加。

### 構造

```
┌─────────────────────────────────┐
│ ▼ 所見                    [↻反映] [▲折りたたみ] │ ← ヘッダー、タップで開閉
├─────────────────────────────────┤
│ ◉ マーカー由来の項目（自動同期、根元→幹→大枝順）│
│   根元：不自然な傾斜                            │
│   根元：巻き根                                  │
│   幹：開口空洞                                  │
│   幹：木槌打診異常（範囲）                      │
│   大枝：枯枝                                    │
├─────────────────────────────────┤
│ ◉ 補足メモ（フリーテキスト）                    │
│ [textarea]                                       │
│ ※マーカーで表現しきれない補足情報を記述         │
└─────────────────────────────────┘
```

### データ構造

樹のJSONに以下を追加（既存の `memo` フィールドはそのまま使う）：

```json
{
  ...
  "memo": "根元：不自然な傾斜\n根元：巻き根\n幹：開口空洞\n幹：木槌打診異常\n大枝：枯枝",  // ← マーカーから自動生成、根元→幹→大枝順
  "memoSupplement": "幹のキャンカー部、約20cm。古い傷の可能性。",  // ← 補足メモ
  ...
}
```

`memo` フィールドは既存どおり、マーカーから自動生成（既存ロジック）。
`memoSupplement` を新規追加。

### マイグレーション

既存樹には `memoSupplement` フィールドがないので、デフォルト `''` で補完：

```javascript
function migrateMemoSupplement(tree) {
  return {
    ...tree,
    memoSupplement: tree.memoSupplement ?? '',
  };
}
```

### 触るファイル

新規：`src/components/ObservationPanel.jsx`（or 適切な名前）
影響：`src/App.jsx`（state管理、マイグレーション）、`src/lib/generateJudgmentReason.js`（memoSupplement 考慮）

## 3. テキストボックス → 所見欄への自動反映

### 仕様

写真上のテキストボックスを編集すると、対応する所見欄のリスト項目も**自動更新**される。

例：
- テキストボックス「開口空洞」を「開口空洞（北側、深さ約10cm）」に編集
- 所見欄の「幹：開口空洞」が自動的に「幹：開口空洞（北側、深さ約10cm）」に変わる

### 実装

`memo` フィールドはマーカーから自動生成されているので、マーカーの `item` が更新されれば自動的に再生成される。

既存の `generateMemoFromMarkers()` 的なロジックがあるはず（写真ファーストフロー実装時に作ったもの）。これに加えて：

#### 順序の固定（根元→幹→大枝）

```javascript
function generateMemoFromMarkers(markers) {
  const partOrder = ['根元', '幹', '大枝'];
  
  // 部位ごとにグループ化
  const grouped = { 根元: [], 幹: [], 大枝: [] };
  for (const m of markers) {
    if (grouped[m.part]) {
      grouped[m.part].push(m.item);
    }
  }
  
  // 部位順に組み立て
  const lines = [];
  for (const part of partOrder) {
    for (const item of grouped[part]) {
      lines.push(`${part}：${item}`);
    }
  }
  
  return lines.join('\n');
}
```

### 動作確認

- [ ] テキストボックスを編集 → 所見欄リストの該当行が即更新される
- [ ] マーカー追加 → 所見欄リストに新しい行が追加される（順序は根元→幹→大枝）
- [ ] マーカー削除 → 所見欄リストから該当行が消える

## 4. 所見欄 → テキストボックスへの手動反映ボタン

### 仕様

所見欄のリスト項目の右側に「反映」ボタンを置く。
所見欄のテキストを直接編集した場合、ボタンを押すまで写真上のテキストボックスには反映されない（誤反映防止）。

### UI

```
所見:
┌─────────────────────────────────┐
│ 根元：不自然な傾斜（北東方向）  [→反映]  │ ← ボタン押すと、対応するマーカーに反映
│ 幹：開口空洞                   [→反映]  │
│ ...
└─────────────────────────────────┘
```

### 操作フロー

1. 所見欄のリスト項目の**テキスト部分をタップ** → 編集モード
2. テキストを変更
3. 「反映」ボタン押す → 該当マーカーの `item` フィールドを更新（部位は変えない）
4. 結果として写真上のテキストボックスも更新される

### 実装注意点

- 編集モード中は「反映」ボタンの色を強調（青ハイライト等）
- 編集後、反映せずに別の場所をタップしたら **編集内容を破棄**（または確認ダイアログ）
- どのマーカーに反映するかは、リスト行と marker.id の対応をひも付けて管理

### 動作確認

- [ ] リスト項目のテキストを編集 → 「反映」ボタンが強調表示
- [ ] 反映ボタン押す → 写真上のテキストボックスも更新される
- [ ] 反映せずに別所をタップ → 編集が破棄される
- [ ] 部位は変えられない（編集できるのは項目名のみ）

## 5. 所見欄のワンタップ開閉

### 仕様

所見欄のヘッダー部分（「所見」と書いてある行）をワンタップで開閉できる。

```
閉じた状態：
┌─────────────────────────────────┐
│ ▶ 所見 (5件)                              │ ← タップで展開
└─────────────────────────────────┘

開いた状態：
┌─────────────────────────────────┐
│ ▼ 所見 (5件)                              │ ← タップで折りたたみ
├─────────────────────────────────┤
│ ... 内容 ...
└─────────────────────────────────┘
```

- デフォルトは**開いた状態**
- 開閉状態は state で管理（永続化は不要、樹を再オープンしたらデフォルト）
- 件数表示：マーカーの数を `(5件)` のように表示

### 実装

```jsx
const [isOpen, setIsOpen] = useState(true);

<div>
  <button
    onClick={() => setIsOpen(!isOpen)}
    style={{ /* ヘッダースタイル */ }}
  >
    {isOpen ? '▼' : '▶'} 所見 ({markers.length}件)
  </button>
  {isOpen && (
    <div>
      {/* リスト */}
      {/* テキストエリア */}
    </div>
  )}
</div>
```

## 6. 所見欄の項目順を根元→幹→大枝で固定

### 仕様

これは項目3「自動反映」の `generateMemoFromMarkers()` で既に対応している（partOrder 配列で定義）。

### 動作確認

- [ ] 「大枝→根元→幹」の順でマーカーを追加 → 所見欄では「根元、幹、大枝」の順に並ぶ
- [ ] 同じ部位の項目は追加順を維持（例：根元の項目を3つ追加したら、追加順に並ぶ）

## 7. undoボタン（直前のマーカー操作を取り消し）

### 仕様

「直前のマーカー操作（追加・削除・編集・移動）を取り消す」ボタンを追加。

```
画面上部にボタン:
┌─────────────────────────────────┐
│ [↶ ひとつ戻る]                            │
└─────────────────────────────────┘
```

### 実装方針：履歴スタック

各樹のマーカー操作の履歴をスタックで管理。

```javascript
const [markerHistory, setMarkerHistory] = useState([]);

function pushHistory(currentMarkers) {
  // 履歴に追加（最大20件まで保持）
  setMarkerHistory(prev => [...prev.slice(-19), currentMarkers]);
}

function undo() {
  if (markerHistory.length === 0) return;
  const previousMarkers = markerHistory[markerHistory.length - 1];
  setMarkerHistory(prev => prev.slice(0, -1));
  // 樹の markers を previousMarkers に戻す
  updateTreeMarkers(previousMarkers);
}
```

### どこで履歴を push する？

マーカーが変更される直前に、変更前の状態を履歴に積む：
- マーカー追加：追加前の markers をスタックに push
- マーカー削除：削除前の markers をスタックに push
- マーカー編集（item変更、位置変更など）：変更前の markers を push
- 範囲マーカーの両端ドラッグ：ドラッグ開始時に push（ドラッグ中の連続更新は1つの操作として扱う）

### UIの細かい仕様

- ボタンは画面上部、樹のタブ近くに配置
- 履歴が空のときはボタンを**無効化（グレーアウト）**
- 押すたびに1ステップ戻る
- 連打で連続undo可能
- redo（やり直し）は今回は実装しない（複雑になるので）

### 制約

- 履歴は樹を切り替えると**リセット**（樹ごとに別管理にすると複雑なので）
- アプリ再起動でも履歴は失われる（メモリ上のみ）
- マーカー以外の操作（樹勢の数値変更、判定ボタンなど）は対象外（マーカー操作だけ）

### 動作確認

- [ ] マーカー追加 → undo → マーカーが消える
- [ ] マーカー削除 → undo → マーカーが復活する
- [ ] マーカー編集 → undo → 編集前のテキストに戻る
- [ ] マーカー移動 → undo → 移動前の位置に戻る
- [ ] 連続undoで複数ステップ戻れる
- [ ] 履歴が空ならボタンが無効化される
