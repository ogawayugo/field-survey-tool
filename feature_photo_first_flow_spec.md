# 写真ファースト・フロー 仕様書

## 概要

街路樹現場調査ツールを「**写真メイン**」のUIに大改造する。現場メモ欄を撤廃し、全景写真へのマーキング操作で診断項目を入力する。

**ブランチ**：`feature/photo-first-flow`
**目的**：現場でのスマホ親指1本完結を実現、現場入力フローを最も自然な形に再設計

## ユースケース

### シナリオ1：新規調査（理想形）

1. 樹に近づく
2. アプリで全景写真を撮る
3. **写真の中の傾斜している根元をタップ** → チップシート開く
4. **「根元」**を選ぶ → 根元用チップ一覧
5. **「不自然な傾斜」**を選ぶ → 確定
6. 画面に青い 📍 マーカーが配置される
7. 次に幹の子実体をタップ → 「幹」→「子実体」→ 緑マーカー
8. クローズアップ写真を3枚撮る
9. 予備の写真もいくつか撮る
10. 樹勢/樹形/活力判定/外観判定/総合判定をボタンで入力
11. 特記事項があれば入力（任意）
12. 「次の樹」へ

### シナリオ2：既存データを開いたとき

旧形式の樹（mema あり、markers なし）を開くと、写真上部に**変換促進バナー**：

> 📝 この樹は旧形式です。
> 現場メモを写真マーカーに変換しますか？  [変換する] [そのまま使う]

- 「変換する」を押す → メモから自動的にマーカー生成（部位ごとにデフォ位置に配置）
- 「そのまま使う」を押す → メモは別欄で読み取り可能、マーカーは新規追加のみ

### シナリオ3：Excelで写真を差し替えたいとき

PCで撮りためた予備写真の中から「やっぱりこっちが良い」となったとき：

1. 写真エリアで**予備写真**を長押し
2. **メイン枠 or クローズアップ枠1〜3**にドラッグ
3. 入れ替わる（元の写真は予備行きへ）

## データ構造

### 樹のJSON構造（B案：互換性最大）

```json
{
  "id": "tree_001",
  "treeNumber": 1,
  "species": "アカガシ",
  "vitalitySei": 2,
  "vitalityKei": 2,
  "vitalityJudgment": "B1",
  "partJudgments": { "根元": "B1", "幹": "B2", "大枝": "B1" },
  "appearanceJudgment": "B2",
  "overallJudgment": "B2",
  
  "markers": [
    {
      "id": "m1",
      "x": 0.42,
      "y": 0.78,
      "part": "根元",
      "item": "不自然な傾斜",
      "createdAt": "2026-05-11T14:30:00Z"
    },
    {
      "id": "m2",
      "x": 0.51,
      "y": 0.62,
      "part": "幹",
      "item": "子実体（ベッコウタケ）",
      "createdAt": "2026-05-11T14:31:00Z"
    }
  ],
  
  "memo": "根元：不自然な傾斜\n幹：子実体（ベッコウタケ）",
  
  "photos": [
    { "id": "p1", "role": "main",     "data": "base64..." },
    { "id": "p2", "role": "closeup1", "data": "base64..." },
    { "id": "p3", "role": "closeup2", "data": "base64..." },
    { "id": "p4", "role": "closeup3", "data": "base64..." },
    { "id": "p5", "role": "spare",    "data": "base64..." },
    { "id": "p6", "role": "spare",    "data": "base64..." }
  ],
  
  "specialNotes": "次回フォローアップ事項"
}
```

### フィールド詳細

#### `markers` 配列

各マーカーはオブジェクト：
- `id`：内部識別子（UUID）
- `x`, `y`：写真上の正規化座標（0.0〜1.0、写真サイズ非依存）
- `part`：`'根元' | '幹' | '大枝'`
- `item`：診断項目の名称（既存のDIAGNOSIS_CHIPSと一致）
- `createdAt`：作成タイムスタンプ

#### `memo` フィールド

**マーカーから自動生成**される。手動編集なし。生成ロジック：

```
markers配列をpart毎にグルーピング:
  根元: "不自然な傾斜"
  幹: "子実体（ベッコウタケ）"

→ memo として：
  "根元：不自然な傾斜\n幹：子実体（ベッコウタケ）"
```

これにより既存の `generateAppearanceReason` ロジック（メモから所見を構築）が**そのまま動く**。PC側のgenerate.pyも書き換え不要。

#### `photos[].role` フィールド

- `main`：全景メイン（マーカー対象、カルテのBM13に焼き込む）
- `closeup1` / `closeup2` / `closeup3`：クローズアップ（AW37/BK37/BY37に焼き込む）
- `spare`：予備（カルテには貼られない、後で差し替え可能）

各役割は**最大1枚**（spare のみ複数）。

### マイグレーション

既存樹データを開くときの処理：

```javascript
function migrateTreeForPhotoFlow(tree) {
  return {
    ...tree,
    markers: tree.markers ?? [],
    photos: tree.photos.map(p => ({
      ...p,
      role: p.role ?? inferRoleFromIndex(tree.photos.indexOf(p)),
    })),
    _legacyMemo: !tree.markers ? tree.memo : undefined,  // 変換確認用
  };
}

function inferRoleFromIndex(index) {
  if (index === 0) return 'main';
  if (index === 1) return 'closeup1';
  if (index === 2) return 'closeup2';
  if (index === 3) return 'closeup3';
  return 'spare';
}
```

## UI仕様

### 画面レイアウト（樹編集画面）

```
┌──────────────────────────────────┐
│ 1 アカガシ                              │
├──────────────────────────────────┤
│                                          │
│        [全景写真（メイン）]              │
│            📍(青)                        │
│              📍(緑)                      │
│                                          │
│   [変換バナー（既存データ時のみ）]       │
│                                          │
├──────────────────────────────────┤
│ [クローズアップ1][2][3]                  │
├──────────────────────────────────┤
│ 樹勢 1 2 [3] 4 5  樹形 1 [2] 3 4 5     │
│ 部位判定（既存どおり）                    │
│ 活力/外観/総合判定（既存どおり）          │
│ 判定理由（既存どおり）                    │
│ 特記事項（残す）                          │
├──────────────────────────────────┤
│ ▼ 予備写真（任意の枚数）                 │
│ [予備1][予備2][予備3]...                 │
└──────────────────────────────────┘
```

### マーカータップ→チップシート（A案）

```
①写真をタップ
       ↓
┌──────────────────────────────────┐
│        [全景写真]                        │
│            🔘 ← 仮マーカー（薄い）       │
│                                          │
├──────────────────────────────────┤  ← 下からせり出すシート
│ 部位を選択                                │
│ [ 根元 ] [ 幹 ] [ 大枝 ]                 │
│                                          │
│ 診断項目（部位選択後に表示）              │
│ [ 不自然な傾斜 ] [ 巻き根 ]              │
│ [ ルートカラー見えない ] [ 露出根 ]      │
│ [ 根元の揺らぎ ] [ 子実体 ]              │
│                                          │
│            [キャンセル] [ 確定 ]         │
└──────────────────────────────────┘
```

選択フロー：
1. **部位ボタン**をタップ（必須）
2. その部位に該当する**診断チップ一覧**が表示される
3. チップを選んで**確定**
4. 仮マーカーが実マーカー（部位色）になる
5. シートが閉じる

### マーカー編集メニュー（既存マーカータップ時）

```
既存📍をタップ
       ↓
┌─────────────┐
│ 部位: 根元      │
│ 項目: 不自然な傾斜│
│                 │
│ [ 編集 ] [ 削除 ]│
└─────────────┘
```

- **編集**：チップシートが再度開き、部位/項目を変更可能（位置は変えない）
- **削除**：確認ダイアログ → 削除

位置を変えたい場合は、削除 → タップしなおし（仕様簡素化のため）。

### 写真ドラッグ&ドロップ

予備写真エリアの写真を**長押し → ドラッグ** → メイン枠 or クローズアップ枠にドロップ。

入れ替わり時の挙動：
- 予備→メイン：旧メインが予備行きへ、旧メインに焼き込まれたマーカーは新メインに移動
- 予備→クローズアップ1：旧クローズアップ1が予備行きへ（マーカー無関係）
- メイン↔クローズアップ：マーカーは混乱を避けるため**メインに残る**（クローズアップ写真を入れ替えるイメージ）

## カルテExcelへの反映

### マーカーから既存マッピングへの変換

generate.py（PC側）は**ノータッチ**。なぜなら、PWA側で`memo`を自動生成しているので、generate.pyの既存ロジック（`extract_diagnosis_item_name`等）がそのまま動く。

ただし、写真焼き込みのロジック（`annotate_photo`）には変更が必要：

#### 旧（v2.8の写真マーキング）
PWA側で手動マーキングした座標を写真に赤ラベル付けて焼き込み。

#### 新（写真ファーストフロー）
markers配列の座標を使って、**黒のテキスト+白フチ**でマーカー位置に診断項目名を焼き込み。

```python
# photo_annotator.py 内
for marker in markers:
    x = int(marker['x'] * img.width)
    y = int(marker['y'] * img.height)
    text = f"{marker['part']}: {marker['item']}"
    
    # 白フチ付き黒テキストを描画
    draw_text_with_outline(
        img, x, y, text,
        text_color=(0, 0, 0),
        outline_color=(255, 255, 255),
        outline_width=2,
    )
```

## Phase分割

実装は5段階に分けて、各Phaseごとに動作確認しながら進める。

### Phase 1：データ構造の準備（軽）

- `markers` フィールドを追加
- `photos[].role` フィールドを追加
- マイグレーション関数の実装
- 既存樹を開いてもエラーが起きないことを確認

**変更なしの確認**：この段階ではUIは変えない。データ層だけ整える。

### Phase 2：写真UIの再構成（中）

- メイン/クローズアップ1〜3/予備の枠を表示
- 各枠への撮影ボタン
- ドラッグ&ドロップで枠間入替
- 予備写真のリスト表示

**まだマーカーは付かない**。写真の整理だけが動く状態。

### Phase 3：マーカー機能（中〜重）

- 全景写真へのタップ検出
- チップシートUI（部位選択 → チップ選択）
- マーカー配置（部位別色分け）
- マーカータップで編集メニュー
- 既存マーカーの編集・削除

### Phase 4：診断データ連携（中）

- マーカー → memo の自動生成
- カルテExcel生成テスト（既存ロジックが動くこと確認）
- 写真焼き込みロジックの更新（黒+白フチ）

### Phase 5：UI整理（軽）

- 現場メモ欄を削除
- 「マーカーに変換」スイッチ実装
- 全体レイアウトの微調整

## やってはいけないこと

- **mainブランチを直接触らない**：必ず feature/photo-first-flow で作業
- **既存データを壊さない**：マイグレーションは必ず後方互換
- **PC側 generate.py を不用意に書き換えない**：memoから自動構築するので既存ロジックを温存
- **全Phase一気に実装しない**：各Phaseで動作確認しながら進める
