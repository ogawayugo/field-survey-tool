# 修正指示書：マーカーUI追加改善（バッジ位置＋双方向ハイライト）

## 概要

写真ファースト・フロー実装後の追加フィードバック2件。

ブランチ：`feature/photo-first-flow`（既存のまま継続）

---

## 改善1：「全景（メイン）」バッジを写真にオーバーレイ

### 現状の問題

写真の下端にあるバッジ「全景(メイン)」と、その横の病害チップ「●キノコ（コフキタケ）」が**同じ行に並んで重なって表示**されている。両方とも灰色〜暗色なので識別もしづらい。

```
[写真]
─────────────
[全景(メイン)] [●キノコ（コフキタケ）] [●キノコ...]  ← 重なって見づらい
```

### 改善後

バッジを**写真の左上にオーバーレイ表示**する。病害チップ群は写真の下に独立した行で並べる。

```
┌─────────────────┐
│[全景(メイン)]      │ ← オーバーレイ、写真左上
│                    │
│      [写真本体]    │
│                    │
└─────────────────┘
[●キノコ] [●傾斜] [●子実体]  ← チップ群はここに独立配置
```

### 具体仕様

- **バッジ位置**：写真の左上角からマージン 8px
- **バッジスタイル**：
  - 背景：半透明黒 `rgba(0, 0, 0, 0.6)`
  - テキスト色：白
  - パディング：縦 4px、横 8px
  - 角丸 4px
  - フォントサイズ：11px（小さめ）
  - 影：軽め
- **z-index**：写真の上、マーカーオーバーレイより下（マーカーが優先）

### 触るファイル

`src/components/PhotoFrameGrid.jsx` または相当のコンポーネント。

メインの全景写真エリアだけでなく、**クローズアップ1/2/3 にも同様に**「クローズアップ1」「クローズアップ2」「クローズアップ3」を左上にオーバーレイ表示すると統一感が出る。

予備写真は「予備」ラベルを右下に小さく表示でもOK（こちらは枚数が多い場合があるので識別必要）。

## 改善2：写真テキストボックスと下部リストの双方向連動

### 現状の問題

写真上のマーカー（テキストボックス）と、下部の病害リスト（チップ）が連動していない。
ユーザーは「これは写真のどこのことを言っているのか」を一目で対応づけできない。

### 改善後

**双方向ハイライト**：

#### 写真上のテキストボックスをタップ
- そのマーカー自体が「選択中」状態になる（枠が太くなる・拡大する等）
- 下部リストの対応する病害チップが**ハイライト**＋**少し上にずれる/拡大する**

#### 下部リストの病害チップをタップ
- そのチップが「選択中」状態（背景色変化など）
- 写真上の対応するテキストボックスが**ハイライト**＋**拡大する** or **アニメーションで揺れる**

### 具体仕様

#### 共有状態

`selectedMarkerId` という state を1つ用意。「今選択されているマーカーのID」を保持。
- 何も選択されていない時：`null`
- 写真側 or リスト側のどちらかをタップ：そのマーカーIDを `selectedMarkerId` にセット
- 同じものを再度タップ：選択解除（`null` に戻す）
- どこか別の場所をタップ：選択解除

これをマーカー一覧コンポーネントと写真オーバーレイの両方で参照する。

#### 写真側のハイライト（テキストボックス）

選択時の見た目変化：
- 枠線が太くなる：2px → **3〜4px**
- スケール：1.0 → **1.05倍**にトランジション
- ボックスシャドウを強める
- z-index を一時的に上げる（他のマーカーより前に表示）

```jsx
const isSelected = selectedMarkerId === marker.id;

<div style={{
  // 既存スタイル
  border: `${isSelected ? 3 : 2}px solid ${partColor}`,
  transform: `translate(-50%, -100%) scale(${isSelected ? 1.05 : 1.0})`,
  boxShadow: isSelected
    ? '0 4px 12px rgba(0,0,0,0.4)'
    : '0 2px 4px rgba(0,0,0,0.2)',
  zIndex: isSelected ? 20 : 10,
  transition: 'all 0.2s ease',
}}>
  ...
</div>
```

#### リスト側のハイライト（病害チップ）

選択時の見た目変化：
- 背景色が薄く部位色になる（例：薄い青／緑／赤）
- 縦方向に少し上にずれる（`translateY(-3px)`）
- 影が強まる

```jsx
const isSelected = selectedMarkerId === marker.id;

<button style={{
  // 既存スタイル
  backgroundColor: isSelected ? lightenPartColor(marker.part) : 'transparent',
  transform: isSelected ? 'translateY(-3px)' : 'translateY(0)',
  boxShadow: isSelected ? '0 4px 8px rgba(0,0,0,0.15)' : 'none',
  transition: 'all 0.2s ease',
}}>
  ● {marker.item}
</button>
```

`lightenPartColor` は部位色を薄くした値を返すヘルパー：
- 根元 `#2563eb` → `#dbeafe`（blue-100）
- 幹 `#16a34a` → `#dcfce7`（green-100）
- 大枝 `#dc2626` → `#fee2e2`（red-100）

### 触るファイル

主に：
- `src/App.jsx` または同等：`selectedMarkerId` の state を上位コンポーネントで管理し、子コンポーネントに渡す
- `src/components/MarkerOverlay.jsx`：写真上のテキストボックスのスタイルに `isSelected` 反映
- 下部の病害リストコンポーネント（おそらく既存に `MarkerList.jsx` 的なものがあるはず、なければ作る）：チップのスタイルに `isSelected` 反映

### 細かい仕様

- 編集モードに入っているマーカーは「選択中」とは別扱い（編集中の見た目を優先）
- 折り畳み中のマーカー（点状態）にも双方向ハイライトは適用（点が少し大きくなる、点滅、など）
- 何もない場所をタップしたら選択解除

## やってはいけないこと

- 既存のマーカー作成・編集・削除の挙動を変えない
- データ構造の変更不要（`selectedMarkerId` は表示用のUI状態のみ、JSONに保存しない）
- main ブランチに直接 push しない（feature/photo-first-flow で作業）

## やっていいこと

- バッジの色やフォントサイズの微調整（実機で見て調整）
- ハイライトのアニメーション時間・量の調整（0.2sは目安、より速く/遅くしてもOK）
